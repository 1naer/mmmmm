# Roadmap

本文档用于记录 mmm V4 的后续完善方向。

## P0：可用性基础

- [x] 完善分享链接识别规则和输入校验
- [x] 补全 `GM_xmlhttpRequest` 异步回调链路
- [x] 为 JSBridge 增加更明确的错误返回结构
- [x] 梳理 WebView 注入时机，避免重复注入和页面跳转丢失状态

## P1：下载能力

- [ ] 下载任务持久化，建议接入 Room 或轻量 SQLite 封装
- [ ] 前台服务下载，适配 Android 13+ 通知权限
- [ ] 下载进度、暂停、恢复、失败重试
- [ ] Range 分片下载的断点续传和文件校验
- [ ] 统一下载目录和文件名冲突处理

## P2：网盘适配

- [ ] 优先适配百度网盘分享页
- [ ] 预留适配器接口，降低不同网盘脚本耦合
- [ ] 支持外部脚本更新或内置脚本版本管理

## P3：性能与体验

- [ ] 下载列表 UI 和任务详情页
- [ ] 网络状态感知与限速策略
- [ ] 大文件下载内存与线程数控制
- [ ] 错误日志导出，便于问题定位

## P4：工程化

- [ ] 增加单元测试和核心下载逻辑测试
- [ ] 增加 lint / detekt 类质量检查，当前为 Java 项目可先强化 Android Lint
- [ ] 配置 Release 签名流程，密钥通过 GitHub Secrets 注入
- [ ] 根据项目定位补充 LICENSE

## 合规边界

- 只处理用户有权访问的文件
- 不内置、传播或鼓励绕过版权与访问控制的行为
- 不收集用户账号、Cookie、Token 等敏感信息，必要时应本地加密保存并提供清除入口
